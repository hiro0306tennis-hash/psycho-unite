# Claude Code 投入プロンプト：エスパー統一・脅威分析HUD

---

## 役割
ポケモンチャンピオンズ（レギュレーションM-B、2026年6月17日開始、9月2日まで）の競技プレイヤー向けに、
**「現環境で最も勝率の見込めるエスパータイプ統一パーティ（6体）」を根拠付きで選定し、
構築シート＋脅威分析ツールを1枚のHTMLにまとめる**こと。

最終成果物は GitHub にプッシュ → Netlify 公開できる単一HTMLファイル。

---

## 大前提（ここを曖昧にしないこと）
- エスパー統一はガチ環境のトップtierではない。「縛りの中での最適解」を出すのが目的。
  勝率を盛らず、不利な点は不利と明記すること。
- 私（依頼者）はデータアナリスト。**結論ファースト・根拠明示・一般論禁止**を守ること。
- 採用した6体それぞれに「なぜ選んだか」「環境の何に対して刺さるか」を1〜2文で必ず添える。

---

## ステップ1：環境調査（手を抜かない）
以下を**必ずweb検索で確認**してから構築を組むこと。記憶で組まない。
1. M-B シングルの使用率ランキング上位20体（=仮想敵）
2. M-B で使用可能なエスパータイプのポケモン一覧（メガ含む）と、その評価ランク
3. 上位構築（ランクマ上位/シングル最終300位など）で頻出する「あくタイプ」「先制技持ち」「鋼タイプ」
   → エスパー統一の天敵カテゴリなので、ここを名指しで列挙する
4. 主要なエスパー駒の種族値・覚える技・推奨持ち物（できれば実数値・SP配分も）

参照候補サイト（鵜呑みにせず複数突き合わせ）：
- yakkun.com（ポケ徹）/ game8.jp / gamewith.jp / altema.jp / appmedia.jp
- 構築コレクション系: pokemon-teams.pages.dev

検索で確証が取れなかった数値は「未確認」と明示し、断定しないこと。

## ステップ2：6体の選定（基準を守る）
選定基準（優先順）：
1. **天敵（あく・先制技・鋼）への回答を最低1〜2枚は積む**
   - 例：あく無効化や、あく/鋼に刺さるサブウェポン（かくとう・ほのお・じめん等）を持つエスパー
2. **役割を分散**：起点作成（妨害）／高速アタッカー／積みエース／受け・詰め筋／天候 or 設置 のうち複数を埋める
3. **メガ枠は1つ**（M-Bは1戦1回のメガ制約あり）。エスパーのメガで最も腐りにくいものを選ぶ
4. 統一なので「全員あく弱点・先制技弱点」を共有する事実を直視し、それでも回る並びにする

各ポケモンについて出力する項目：
- 名前 / 持ち物 / 性格 / 特性 / 技4つ / SP（努力値）配分の方針 / 役割 / 採用理由（環境の何に効くか）

---

## ステップ3：脅威分析（対面表）
- ステップ1で出した使用率上位の仮想敵（最低15体）に対して、
  この6体の並び全体が **有利 / 五分 / 不利** のどれかを判定し、表にする
- 「不利」セルには、どう捌くか（引き先・誤魔化し方）を一言添える
- パーティ全体の**構造的弱点トップ3**を明記（例：◯◯統一は△△に一貫、など）

---

## ステップ4：HTML化（デザイン指定）
- **Psycho-Pass風 / サイバーHUDデザイン**（既存作 "OPERATION/TAKASHI" のトーンに寄せる）
  - 暗背景、シアン/マゼンタのアクセント、等幅フォント混じり、スキャンライン or グリッド演出
  - 過剰演出で可読性を落とさない。「見た瞬間に判断できる」が最優先
- 構成（上から）：
  1. ヘッダー：パーティ名（例 OPERATION: PSYCHO-UNITE）／レギュ表記／総合評価（S〜D）
  2. 6体の構築カード（横並び or グリッド）。各カードにステップ2の全項目
  3. 脅威分析テーブル（仮想敵 × 有利不利。色で即判断できるよう緑/黄/赤）
  4. 構造的弱点トップ3 と 立ち回り指針
  5. フッター：データ取得日（今日の日付）／参照レギュ／"data current as of M-B" 注記
- 単一HTMLファイル（CSS/JS内包、外部依存なし）。スマホでも崩れないレスポンシブ
- localStorage等のブラウザストレージは使わない（不要）
